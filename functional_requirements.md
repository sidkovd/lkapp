## ЭКРАН 1: Авторизация по телефону

### Сценарий использования
Пользователь открывает приложение → вводит номер телефона → получает OTP с кодом → вводит код → попадает в главный экран

### Состояния экрана

#### 1.1. Ввод номера телефона
**Элементы:**
- Логотип обменника
- Заголовок: "Вход в личный кабинет"
- Подзаголовок: "Введите номер телефона"
- Поле ввода телефона:
  - Маски может быть 2. 
(если номер РФ) Маски: `+7` и `(___) ___-__-__`
(если номер Тай) Маски: `+66` и `__ ___ ____`
соответственно нужен селектор/переключатель
  - Автоматическое форматирование при вводе
  - Плейсхолдер: "+7 (___) ___-__-__"
- Кнопка "Продолжить":
  - Неактивна, пока номер не введён полностью
  - Активна при корректном формате
- Текст внизу: "Нажимая «Продолжить», вы соглашаетесь с [Политикой конфиденциальности] и [Условиями использования]"
- Ссылки на политику и условия (кликабельные)
- Текст внизу виден даже при активной клавиатуре.
- Иконка очистить телефон справа от номера. (появляется только при введенных символах)

**Валидация:**
- Проверка формата номера
- Визуальная индикация ошибки (красная рамка, текст ошибки)

**Состояния кнопки:**
- Disabled: номер не введён или неверный формат
- Enabled: номер введён корректно
- Loading: отправка запроса на сервер

---

#### 1.2. Ввод кода подтверждения
**Элементы:**
- Иконка крестика или стрелки назад (сверху слева)
- Заголовок: "Введите код для входа"
- Подзаголовок: "Отправлен на номер +7 (XXX) XXX-XX-XX (для тайского другой номер)")
- Поле ввода кода:
  - 6 отдельных полей для цифр
  - Автоматический переход между полями
  - Автоматическая отправка при вводе последней цифры (опционально)
- Таймер: "Отправить код повторно через 00:XX"
  - Неактивная ссылка до истечения таймера
  - Активная ссылка "Отправить код повторно" после истечения
- Кнопка "Подтвердить" (если не автоподтверждение)
- Иконка "Изменить номер" (возврат к экрану 1.1)

**Валидация:**
- Проверка формата кода (только цифры)
- Визуальная индикация ошибки
- Блокировка после 3-5 неудачных попыток

**Состояния:**
- Ожидание ввода кода
- Проверка кода (loading)
- Ошибка: неверный код
- Ошибка: код истёк (предложение отправить новый)
- Успех: переход на главный экран

## ЭКРАН 2: Главная страница

#### 2. Главный экран
### Сценарий использования
Главный экран, задача которого иметь быстрый доступ к самому часто используемому функционалу, калькулятору обмена и оплате тайских QR. Также видеть курсы валют. Курс валют зависит от выбранного офиса [ Пхукет, Централ, Бангкок, Паттайя ].
В калькуляторе должен быть блок с выбором способа получения денег. 

**Элементы:**
- Калькулятор обмена,
- Выбор способа получения денег
- Хоткей [ оплатить ThaiQR ], 
- Табы с выбором офисам [ Пхукет, Централ, Бангкок, Паттайя ] 
- Курсы валют (+ возможность раскрыть полный список)
- Блок с новостями (формат сториза думаю) 
- Страница с новостью (поп-ап с картинкой думаю. текста до 250 слов (без скрола чтобы))

**Валидация:**
- Все инпуты только целочисленные
- Раскрывающийся список валют должен содержать флаг + название валюты
- По умолчанию введена сумма 100 тыс руб для обмена

**Кнопки:**
- Обменять 
- Выбор способа получения (по умолчанию офис)
- Все валюты (развернуть/свернуть)

---

#### 2.1. Выбор способа получения денег
### Сценарий использования
Главный экран, задача которого иметь быстрый доступ к самому часто используемому функционалу, калькулятору обмена и оплате тайских QR. Также видеть курсы валют. Курс валют зависит от выбранного офиса [ Пхукет, Централ, Бангкок, Паттайя ].
В калькуляторе должен быть блок с выбором способа получения денег. 

**Элементы:**
- Выбор способа получения денег с мини описанием:
  - Офис
  (Вы можете забрать наличные в любом их 8ми наших офисов.)
  - ATM (банкомат) (лейбл - без банк. карты)
  (Тайские банкоматы поддерживают функцию удаленной выдачи денег без карты, функция называется Cardless. дословно "без карты")
  - Перевод 
  (Переведем THB на счет в тайском банке.)
  - Доставка (лейбл - от 50 тыс. THB)
  (Доставим в течении 2х часов в любую точку острова.)

**Кнопки:**
- Назад
- Выбор способа получения идет нажатием на соответствующий элемент

---

#### 2.2. Экран потверждения обмена и выбор способа оплаты
### Сценарий использования
Пользователь видит информацию по выбранному обмену. Может вернуться назад. Выбирает способ оплаты

**Элементы:**
- Возврат назад
- Выбор способа оплаты:
  - Перевод по номеру телефона
  Перевод физическому лицу по номеру телефона. 
  - Оплата по QR (лейбл - требуется верификация - линк на профиль) (если вериф есть то лейбла нет)
  Оплата по QR через ваш банк. 
- Начать обмен
