# Функциональные требования для дизайнера

## Общие требования

### Технические ограничения
- Платформа: iOS и Android (PWA + TWA)
- Ориентация: портретная
- Цветовая схема: светлая тема

### Принципы UX
- Минималистичный интерфейс
- Быстрый доступ к основным операциям
- Понятная навигация
- Информативность без перегруза

## ЭКРАН 1: Авторизация по телефону

### Сценарий использования
Пользователь открывает приложение → вводит номер телефона → получает OTP с кодом → вводит код → попадает в главный экран

### Состояния экрана

#### 1.1. Ввод номера телефона
**Элементы:**
- Логотип обменника
- Заголовок: "Вход в личный кабинет"
- Подзаголовок: "Введите номер телефона"
- Поле ввода телефона:
  - Маски может быть 2. 
(если номер РФ) Маски: `+7` и `(___) ___-__-__`
(если номер Тай) Маски: `+66` и `__ ___ ____`
соответственно нужен селектор/переключатель
  - Автоматическое форматирование при вводе
  - Плейсхолдер: "+7 (___) ___-__-__"
- Кнопка "Продолжить":
  - Неактивна, пока номер не введён полностью
  - Активна при корректном формате
- Текст внизу: "Нажимая «Продолжить», вы соглашаетесь с [Политикой конфиденциальности] и [Условиями использования]"
- Ссылки на политику и условия (кликабельные)
- Текст внизу виден даже при активной клавиатуре.
- Иконка очистить телефон справа от номера. (появляется только при введенных символах)

**Валидация:**
- Проверка формата номера
- Визуальная индикация ошибки (красная рамка, текст ошибки)

**Состояния кнопки:**
- Disabled: номер не введён или неверный формат
- Enabled: номер введён корректно
- Loading: отправка запроса на сервер
